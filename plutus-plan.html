01

Set expectations before we set up

The research came back with one loud, consistent message. It doesn't kill the project — it decides how we frame it.

65–79%
of professional active funds trail the S&P 500 over a single year (SPIVA).
Most
frontier LLMs failed to beat buy-and-hold in STOCKBENCH (Oct 2025) — incl. GPT-5, Claude 4, o3.
n≈20–40
decisions in a month — far below the ~30 min / 100+ needed to tell skill from luck.

Beating the market over one month is close to a coin flip — even for professionals with vastly more resources. The newest LLM-specific benchmark found that being good at financial Q&A does not translate into good trading. And a single Sharpe or return figure over ~20 trading days has enormous error bars.

So Plutus is framed as an experiment in how an LLM makes and justifies decisions — not "an AI that beats the market." The headline output isn't the return; it's Plutus's reasoning log plus a rigorous, honest head-to-head against baselines. Sources: S&P Dow Jones SPIVA · STOCKBENCH arXiv:2510.02209 · Harvey & Liu, "Backtesting"

Concretely, this shapes three things: we run multiple bots side-by-side (a lone number means nothing), we pre-register the rules before day one, and every metric on the dashboard is labelled by how much we can trust it at this sample size.

03

Architecture — serverless & free

The GitHub repo is the database. No backend, no hosting bill, and the commit history becomes a tamper-evident audit trail.

Cash + current holdings, per bot. The live snapshot.
trades.json

nav_history.json
One NAV point per run, per bot → drives the equity curve.
  • Market data: Finnhub (60 req/min free) or Twelve Data (8/min, 800/day), fetched in the Action so the key stays secret and every run has one canonical, logged price. Avoid Alpha Vantage (now 25 req/day).

  • Safety valve: if a price fetch fails, log and skip trading rather than commit a corrupt NAV — the month's curve stays trustworthy.

04

The four contestants

Everything held constant except the decision logic — same $1,000, same universe, same cadence, same fee model. Only the brain differs.

Plutus
LLM agent — Opus 4.8 / persona by Fable

Researches the live web each cycle, may trade any US-listed stock or ETF (watchlist is a convenience, not a cage), and must justify every trade in writing. Goal: make money.

VOO Buy & Hold
Passive benchmark

Buys the market on day one and never trades. The bar everything is measured against.

Momentum Bot
Mechanical rule

Ranks the universe by trailing return, holds the top names, rebalances on the same cadence.

Random "Monkey"
Chance baseline

Picks names at random, matched to Plutus's position count and holding period. The luck control.

05

What makes it credible

Two moves do most of the work of separating a real experiment from a fishing expedition.

Pre-registration

Freeze the full rule set in a timestamped commit before day one: the universe, starting capital, cadence, long-only / no-leverage / no-shorting, per-position cap (~15%), cash buffer (~5–10%), and the exact fee + slippage model — applied identically to every bot. Then commit to reporting the outcome whether Plutus wins or loses.

The Monte-Carlo "monkey" test

A single random bot is one draw. Instead we simulate ~5,000 random trade sequences matched to Plutus's trade count and holding period, then plot the distribution of returns. Plutus only "beats random" if it lands in the tail — and the dashboard states the percentile plainly: "Plutus beat X% of random monkeys." This is the honest way to ask skill, or luck?

Guardrails, applied to all bots: long-only, no leverage, no shorting, a fixed liquid universe snapshotted at the start, per-position and per-trade caps, and a cash buffer. Fractional shares allowed (it's simulated), so $1,000 can spread across several names. Sources: Bailey & López de Prado (Probabilistic Sharpe / MinTRL) · "Seven Sins of Quantitative Investing" · RobotWealth random-baseline method

08

Decided

The open questions from v1, now locked and implemented.

Run times — 30 14,19 * * 1-5 UTC
~10:30am and ~3:30pm ET on weekdays. GitHub may delay a few minutes under load; the exact fetch timestamp is logged either way.
Fee & slippage — 5 bps + 5 bps per side
Applied identically to every bot, so churn is penalized fairly. Visible in the very first dry run: VOO's NAV dropped to $999.80 on its one buy.
Opus invocation — direct Anthropic API
Called inside the Action with a zero-dependency HTTP client. Two attempts, then the bot holds. Its market_view is journaled every cycle, traded or not.
Universe — 27 names, frozen
The list in §06 as built, pre-registered in data/config.json. Data: Finnhub primary, Yahoo keyless fallback (Stooq is now behind an anti-bot wall).
Plutus roams; baselines stay caged
Per the owner's brief, Plutus may trade any US-listed name (validated by a live quote at order time) and researches the web (server-side search, ≤6 uses/cycle) before deciding. The fixed universe still binds momentum and random — an accepted asymmetry: Plutus's goal is to make money, and the VOO benchmark is unaffected.
Auto-stop — 21 trading days
The engine freezes trading after the pre-registered window; NAV keeps being recorded. The experiment can't quietly run long because it happens to be winning.
Not investment advice. Plutus trades imaginary money with simulated fills. A one-month, ~20–40-decision run is a single noisy draw from a distribution — a random bot could plausibly match or beat it over this window. Past performance does not indicate future results. This is a demonstration of process, not evidence of edge.
